# Migration Guide — Bash to Rust CLI

This document maps the original Bash scripts to their Rust replacements and highlights any behavioral differences.

## Quick Start

Install the Rust binaries:

```bash
cargo install --path crates/sldo-plan
cargo install --path crates/sldo-run
```

Or build the full workspace:

```bash
cargo build --workspace --release
# Binaries at target/release/sldo-plan and target/release/sldo-run
```

## Behavioral Differences

| Area | Bash | Rust | Impact |
|---|---|---|---|
| Help output format | Custom `usage()` heredoc | clap auto-generated | Cosmetic only — same flags listed |
| Error messages | `echo -e` with colour codes | `colored` crate via stderr | Same colour scheme (red/yellow/green/blue/cyan) |
| Log directory | `.copilot-logs/` | `.copilot-logs/` | Identical |
| Git safety | `git rev-parse` | `git rev-parse` (via `std::process::Command`) | Identical behavior |
| Build/test detection | Bash `if` chain | Rust function with same checks | Same project types detected (Cargo, npm/pnpm/yarn, Go, Make, Python) |
| Copilot invocation | `copilot -p <prompt> --model <model> ...` | Same command via `std::process::Command` | Identical flags passed |
| Template fallback | Inline heredoc in script | Compile-time `const` string | Same template content |
| Cross-platform | macOS/Linux with GNU coreutils | Single compiled binary per platform | Rust is more portable |

## When to Switch

- **Now**: The Rust binaries are feature-complete and fully tested. You can start using them immediately.
- **Bash scripts preserved**: The original scripts in `src/` are not modified and remain functional. Use them as a fallback if needed.
- **Recommendation**: Use the Rust binaries for all new work. They provide better error handling, type safety, and cross-platform support.

## Troubleshooting

### macOS Gatekeeper

On macOS, running a locally compiled binary for the first time may trigger a security prompt. Use `cargo run` as a workaround:

```bash
cargo run -p sldo-plan -- <prompt-file> <repo-dir>
cargo run -p sldo-run -- <runbook> <repo-dir>
```

### Missing `copilot` CLI

Both Bash and Rust versions require the GitHub Copilot CLI to be installed and on your `PATH`. See [GitHub Copilot CLI documentation](https://docs.github.com/en/copilot/concepts/agents/copilot-cli) for installation instructions.
