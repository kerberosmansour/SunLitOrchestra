# Reversibility — kani-verification

| Decision | Why hard to change | Reversibility tactic | Rollback / migration path | Proof required |
|---|---|---|---|---|
| `kani_required` frontmatter key added to `<slug>-overview.md` | Once `/slo-architect`, `/slo-plan`, and `/slo-kani` read it, removing it breaks the handoff contract | Default-when-absent is `false`, so the key is purely additive; old overviews stay valid | Stop emitting the key; readers fall back to `false`. No data migration. | A design overview lacking the key still passes the frontmatter type-check and `/slo-plan` treats it as `false` |
| Pinned `kani-verifier` version in `tools.toml` | A bump may change `cargo kani` flags / output anchors the run/triage parser relies on | Pin is a single line; bump deliberately with a re-verify of the demo crate | Revert the pin; re-run `cargo kani setup`. Record old+new version in the M5 CI doc. | Demo crate (M4) still goes catch→remediate→green on the new pin before the bump merges |
| §5 Kani sub-block added to the v4 runbook template | Template is the output contract of `/slo-plan`; structural tests may assert its shape | Sub-block is additive under the existing §5 (TLA+ content untouched); `N/A` is a valid fill | Remove the sub-block; existing runbooks with `N/A` Kani sections remain valid | A CRUD runbook with `§5 Kani: N/A — no Rust kernels` passes `/slo-plan` and `/slo-retro` |
| Host-neutral skill (vs. Claude-only) | `discover_skills()` installs every `skills/*/SKILL.md`; demoting to Claude-only means moving the dir | Skill is self-contained under `skills/slo-kani/` like `/slo-tla` | Move to `agents/` or gate it; update the catalog + LOOPS doc | `sldo-install --dry-run` lists `slo-kani` among installed skills |
| Demo crate in a **separate repo** | Cross-repo: the failure-bar evidence lives outside this repo's CI | Reference the repo by URL + pinned commit in the M4 verify doc | If isolation proves painful, migrate the crate into `examples/` (the rejected alternative) with a workspace-exclude | M4 verify doc cites the external repo commit SHA exhibiting catch→remediate→green |

No decision here is one-way-door: every seam is additive with a documented fallback. The only genuinely hard-to-reverse cost is *user trust* if a green Kani claim is later shown to have overclaimed scope — which is precisely why the honesty gates (M2) are non-negotiable, not the reversibility tactics.
