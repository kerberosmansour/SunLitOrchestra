# Reversibility — innovation-loop

Hard-to-change decisions, why each is hard to reverse, the reversibility tactic,
the rollback / migration path, and the proof required.

| Decision | Why hard to change | Reversibility tactic | Rollback / migration | Proof required |
|---|---|---|---|---|
| Experiment Book path `docs/slo/experiments/<slug>/EXPERIMENT.md` | All 8 skills target it; once founders have authored real Books, renaming orphans them | Mirror the v4-runbook path convention so it reads as "obvious"; keep the directory shallow | A path move is a one-time `git mv` + a sed of section anchors; skills reference sections by heading, not absolute path beyond the root | A dogfood Book opens, gets filled across phases, and closes — proving the path + section anchors are stable |
| Experiment Book section order (§0–§11) frozen | Skills find their section by heading; reordering breaks resume/append | Number the sections; skills match on the numbered heading | Adding a section is additive (append before §11 Compost); reordering is a migration with a template-version bump | Template version `v1` in the filename; a `v2` would be a new template file, leaving `v1` Books valid (the runbook-v3→v4 precedent) |
| Frozen 8-state exit vocabulary | `/slo-curate` + `/slo-demo` + the tracker + any future reporting key off the exact strings | Centralise the list in the interfaces doc + template §1; skills cite it, never invent | Adding a state is a template + skills + interfaces edit in lockstep (the same 4-touch lockstep the secure-value-loop used for version bumps) | The structural-contract test (or a doc test) asserts the template lists exactly the 8 states |
| 8 separate skills vs. one mega-skill | Once published + installed, splitting/merging changes the command surface founders type | Ship them as the full set now (founder chose the full loop); each is independently invocable and resumable via the shared Book | Merging two phases = deprecate one `SKILL.md`, fold its section-fill into the other, leave a redirect note; the installer drops a removed directory cleanly | Each skill installs (presence of `SKILL.md`) and fills only its own section without clobbering others — proven by a full sandbox→demo dogfood |
| Scratch code lives at `experiments/<slug>/<spike-id>/` (repo root) and is git-IGNORED by default | If tracked, dead-end scratch pollutes history and invites prototype-becomes-production drift; if ignored, evidence isn't shareable | Default: add `experiments/` to `.gitignore`; the *durable evidence* (metrics, screenshots, notes worth keeping) is copied into `docs/slo/experiments/<slug>/evidence/` which IS tracked | Flipping a specific spike to tracked is a deliberate per-spike `git add -f` during promotion, recorded in the Spike Card's delete-or-promote decision | A spike runs, leaves scratch under `experiments/`, and `git status` shows it ignored while its distilled evidence is tracked under `docs/slo/experiments/<slug>/evidence/` |
| `ai_component: true` (AI tolerance contract required) | Drives `/slo-plan` §5B + `/slo-verify` checks; flipping to false later removes the evidence-fabrication / prompt-injection guards | Set true now; the guards are cheap and the agent surface is real | Flipping to false requires re-running `/slo-architect` and justifying why the LLM surface no longer needs tolerance gates | The threat model names ≥3 concrete AI-surface abuse cases (abuse-3, abuse-4, abuse-6) — proof the flag is earned, not decorative |
| No new Rust crate / no service surface | Re-introducing a binary later means a new workspace member + CI + release plumbing | Keep everything Markdown + one test appended to an existing `xtasks` crate | If a deterministic computation is ever needed, it is a *new* runbook adding a crate — additive, not a reversal of this one | `cargo test -p sldo-common -p sldo-install -p sldo-research` plus the sast-verify gate stay green with only a test added |
