# Critique — sast-rulegen-skill-pack (Runbook A)

Four personas rotated in order. Findings below; each row gets the user's accept/decline decision before `/slo-execute M1` runs.

## CEO persona — scope challenge

**Result: no findings.** The wedge is right-sized, the milestone count is at the cap (3 of 5 max), each milestone has a clear user outcome, and the priority order (bootstrap → extend → CI) is the only sane sequence given that M2's value depends on M1's gate. The opportunity to defer M3 was considered (~3 days saved) but rejected: M3 ships LICENSE (without which no downstream consumer can adopt) plus the cargo-audit-driven extend trigger documentation (without which the recurring-value loop has no canonical starting point). Holding scope.

| eng-1 | eng | ask | M1 — `check-clean` BDD | `check-clean` runs against the host crate's `src/`. But `check-clean` treats the firing as a false-positive and exits 2; gate rejects the rule even though the rule is the one finding the developer's actual bug. The host-`src/` scan becomes an opt-in flag for "I want to find the actual bugs that already exist." Add BDD `check_clean_uses_fixture_dir_by_default` and `check_clean_can_target_host_src_with_explicit_flag`. |

| eng-4 | eng | ask | M2 — tier-detect BDD | `tier_detect` BDD covers exactly one `git remote get-url origin` URL shape (HTTPS to github.com). The tier-detect helper's URL-parser doesn't match `https://`, returns `Tier::Unknown` or panics. The skill either crashes or — worse — defaults to `Tier::Public` (the LESS conservative default) and the founder's bug-summary lands tracked in a public repo. | Add BDD scenarios `tier_detect_handles_ssh_url`, `tier_detect_handles_no_remote`, `tier_detect_handles_multiple_remotes`, `tier_detect_handles_non_github_remote`. | eng-5 | eng | ask | M2 — atomic-write contract | The "atomic-write" contract for the 3-5 generated rules is prose-only ("either all 3-5 land or none"). The skill's rollback path is not specified; if the implementation `fs::write`s rules incrementally and only rolls back on the 5th failure, an interrupt (Ctrl-C) between the 4th write and the 5th-rule gate-check leaves 4 ungated rules on disk.
