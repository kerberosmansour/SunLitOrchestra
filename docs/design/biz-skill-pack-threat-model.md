## System description

~~~text
biz-skill-pack adds 12 Markdown skills under skills/slo-<name>/ plus shared scaffolding under references/biz/ at the repo root. Four advisor skills (legal, accounting, equity, fundraise) operate with draft/translate/triage/prepare modes and refuse to draft when matters cross hard-coded gates: regulated domain (FCA/MHRA/ICO/health/finance), deal value > £5,000, counterparty has a lawyer or founder is being asked to sign their paper, and ALL GDPR documents (privacy notice/ROPA/DPA/internal data-protection policies). Eight generator skills (talk-to-users, launch, sales-funnel, pricing, metrics, cofounder, hire, founder-check) produce one artifact each.

UK only in v1. Output convention: docs/biz/ for confidential drafts (gitignored), docs/biz-public/ for placeholder-only artifacts (git-tracked). oneNDA template body is rendered byte-verbatim under CC BY-ND 4.0; the SHA-256 is hash-checked at install time. Skills do NOT enable WebFetch/WebSearch — external regulatory anchors are URL citations the founder follows manually.

Trust boundaries: Founder ↔ Claude Code (founder pastes deal-sensitive info incl. real persons' names); Skill ↔ Founder's repo filesystem (two-tier output); Skill ↔ oneNDA canonical bytes (verbatim render under license); Triage gate ↔ skill behavior (single source of truth in references/biz/triage-gate.md).
~~~

## Top risks (from idea doc)

~~~text
1. Breach: A founder runs `/slo-legal draft contractor-sow` and fills placeholders with real counterparty name, deal value, and IP scope. The output lands in docs/biz/legal/<contractor>-sow.md. If that repo is later pushed to a public GitHub remote, the filled template leaks deal-sensitive terms to anyone running GitHub code search for the standard contractor agreement title plus the founder's company name.

2. Compliance fine: Founder runs `/slo-legal draft privacy-notice` for a B2C app processing personal data. Skill produces an OK-looking notice citing legitimate-interest as the lawful basis. Founder ships without lawyer review. An ICO complaint reveals consent was the correct lawful basis under UK GDPR Art. 6, Art. 13 transparency obligations are not fully met. ICO can fine up to £17.5M or 4% of annual turnover. Mitigation: `draft` mode is hard-blocked for ALL GDPR-related documents.

3. Silent-wrongness on triage: Founder asks `/slo-legal triage "I'm hiring my first employee in UK"` expecting a refuse-and-route response. Skill misclassifies the question as contractor-shaped and offers to draft a contractor agreement, missing UK employment law statutory protections. Founder uses the wrong template; engagement is later reclassified by HMRC as employment under IR35; founder owes back NI + pension contributions and faces ACAS unfair-dismissal exposure. Detection lag: 6–24 months.
~~~

## AI / LLM-specific threats

The biz pack embeds Claude reasoning in every advisor skill. The MITRE ATLAS + OWASP LLM Top 10 + NIST AI RMF triad applies on top of the inherited project-wide controls.

| Technique / risk | ATLAS id | OWASP LLM | NIST AI RMF | Surface in this system | Control |
|---|---|---|---|---|---|
| Prompt injection via founder-supplied prose | AML.T0051 (family) | LLM01 | Measure 2.6 | Founder pastes a contract / pitch / transcript; embedded "ignore previous instructions" tries to bypass the triage gate | Skill prompt asserts the triage gate is enforced regardless of pasted content; `/slo-critique` security persona reviews advisor SKILL.md for injection-resistance language |
| AI agent context poisoning | AML.T0058 | LLM03 | Manage 3.2 | Tampered `references/biz/triage-gate.md` between SKILL invocations | `references/biz/` is in the SLO repo, not user-writable mid-session; `sldo-install` manifest re-compare on re-install detects drift |
| Sensitive info disclosure | — | LLM06 | Measure 2.10 | Founder pastes counterparty names + deal values + IP scope; advisor skill writes them verbatim to `docs/biz/<area>/` | Two-tier output convention + `.gitignore` default; write-time warning on tracked-and-remote path |
| Over-reliance on triage output | — | LLM09 | Govern 3.2 | Founder takes a "below-gate" triage decision as gospel without consulting a professional on a borderline case | Every advisor `triage` output frontmatter sets `lawyer_review_recommended: true` for any borderline; skill prose includes mandatory "this is a triage, not advice" disclaimer; the advisor pattern itself is the recognition that some judgments are LLM-native and others are not |
| Insecure output handling | — | LLM02 | Measure 2.11 | LLM emits markdown that contains injected `<script>` tags or shell-execution prompts; downstream tooling renders unsafely | Outputs land as plain Markdown; SLO has no rendering surface; founder's downstream rendering (e.g. GitHub) sanitises by default |
| Improper output filtering — license violation (oneNDA) | — | (not LLM-specific; license risk) | Govern 5.1 (third-party content) | LLM "improves" the oneNDA body during draft mode | SHA-256 hash check + `/slo-verify` regression test on rendered NDA body bytes |
| Excessive agency | — | LLM08 | Manage 3.1 | Advisor skill draft mode produces a doc the founder signs without realising they crossed a hard-block gate | Frontmatter `triage_gate_passed: bool` and `lawyer_review_recommended: bool` are required and visible in every drafted doc; skill writes "LAWYER REVIEW RECOMMENDED" header at the top of any `draft` artifact regardless of gate status |
