# Rule YAML manifest schema

Read by `cargo xtask sast-verify check-coverage`. Strict-parsed via `serde_yaml_ng` with `deny_unknown_fields`.

## Required structure

```yaml
rules:
  - id: <kebab-rule-id>           # MUST be unique across the rule pack
    languages: [rust]              # v1 = rust; Runbook B adds typescript
    severity: WARNING | ERROR | INFO
    message: <one-line description, ends with a period>
    metadata:                      # REQUIRED block
      cwe: "CWE-<NNN>: <Title>"   # MUST start "CWE-" + numeric id from cwe-map-rust.md
      category: security
      confidence: HIGH | MEDIUM | LOW
      source-of-bug-shape: <RUSTSEC-id> | clippy-<lint> | manual
      sldo-rulegen-version: <git-sha-of-skills/slo-rulegen/SKILL.md-at-emit>
      sldo-variation-template: references/sast/variations/cwe-<NNN>.md
    pattern-either:
      - pattern: <variation 1>
      - pattern: <variation 2>
      - ...
    pattern-not-inside: |
      #[cfg(test)]
      mod tests { ... }
```

## Field rules

- **`id`** — kebab-case, prefixed with the CWE id for browse-ability (e.g., `cwe-755-panic-on-result-fn`, `cwe-416-uaf-after-vec-grow`). MUST be unique across `.semgrep/<lang>/`.
- **`languages`** — exactly one element for v1: `[rust]`. Runbook B adds `[typescript]` rules to a separate `.semgrep/typescript/` tree.
- **`severity`** — `WARNING` is the default for v1. `ERROR` reserved for direct memory-safety classes. `INFO` is permitted only for advisory rules that should not block CI; document why.
- **`message`** — present-tense, one sentence, ends with a period. Must name the bug class, not just the pattern.
- **`metadata.cwe`** — MUST match `^CWE-\d+: .+$`. The CWE id MUST appear in `references/sast/cwe-map-rust.md`. Custom CWE ids are forbidden.
- **`metadata.confidence`** — HIGH = pattern-fires-only-on-real-bug. MEDIUM = pattern-fires-on-bug-class-but-may-FP. LOW = advisory; review-driven only.
- **`metadata.source-of-bug-shape`** — provenance trail. RUSTSEC-id when the variation came from an advisory diff; `clippy-<lint>` when the sink shape was inspired by a Clippy restriction lint; `manual` when the author derived the shape independently.
- **`metadata.sldo-rulegen-version`** — emitted by the skill at gen-time. Lets `/slo-ruleverify` flag rules generated by older skill revisions for re-validation when the rule-pack format evolves.
- **`metadata.sldo-variation-template`** — path to the variation template doc. Allows `check-coverage` to load the per-CWE minimum and `cwe_<NNN>_rule_covers_documented_variation_shapes` to check sink-shape content coverage.
- **`pattern-either`** — REQUIRED when the variation template's `minimum_pattern_either_arms` > 0. Single-pattern rules (without `pattern-either`) are exempt only when the variation template explicitly declares `minimum_pattern_either_arms: 0`.
- **`pattern-not-inside`** — recommended for any rule that could fire on test code. Use the canonical exclusion shape `#[cfg(test)] mod tests { ... }`.

## Forbidden fields

- Inline `metadata.examples` — examples live in the paired `<rule-id>.rs` fixture, not in YAML.
- `metadata.tags` (Semgrep custom tags) — adds parser surface; we use `cwe` and `confidence` instead.
- Any `vendor.*` namespaces beyond the `sldo-*` prefix.
- Raw URLs in `pattern:` arms — would defeat the prompt-injection-resistance posture (sec-2). Patterns are AST shapes only.

## Disallowed primitives for v1

- `metavariable-type` on Rust generics — partial support per Semgrep open issues #10380 and #11150. Use structural `pattern-inside fn $F(...) -> Result<$T1, $T2> { ... }` instead.
- `pattern-inside: unsafe { ... }` — UNVERIFIED in 2026 Semgrep CLI. M1 smoke-test in `references/sast/semgrep-rust-syntax.md` confirms or invalidates. Until smoke confirms, use the workaround: `pattern-inside: fn $F(...) { ... unsafe { ... } ... }`.
- Taint mode (`mode: taint` with `pattern-sources` / `pattern-sinks`) — works for Rust per the research dossier, but constrained for v1. Reach for taint only when structural enumeration genuinely fails.

## Validation chain

1. `serde_yaml_ng::from_str` with `deny_unknown_fields` (xtask `validate` step 1).
2. `semgrep --validate --json` for deeper rule-shape check (xtask `validate` step 2).
3. `check-coverage` reads `metadata.cwe` → loads `variations/cwe-<NNN>.md` → asserts pattern-either arm count.

A rule that fails any of the three is rejected by `gate`.
