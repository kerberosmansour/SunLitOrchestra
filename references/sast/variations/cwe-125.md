# Variation template — CWE-125 Out-of-bounds Read

## Why this class

Safe Rust panics on `slice[i]` past the end. OOB reads surface in `unsafe` slice access, decompression / parser code that trusts attacker-supplied length fields without bounds-checking, and `slice::from_raw_parts` with attacker-controlled lengths.

## Sink shapes (3 minimum)

### 1. `unchecked_get_unchecked`

```rust
fn read(buf: &[u8], i: usize) -> u8 {
    unsafe { *buf.get_unchecked(i) }   // BAD: no bounds check
}
```

### 2. `slice_from_raw_parts_attacker_len`

```rust
unsafe fn read_packet(ptr: *const u8, len_from_attacker: usize) -> &'static [u8] {
    std::slice::from_raw_parts(ptr, len_from_attacker)   // BAD if len untrusted
}
```

### 3. `parser_reads_attacker_length_field_unbounded`

```rust
fn parse_chunk(buf: &[u8]) -> &[u8] {
    let claimed_len = u32::from_le_bytes(buf[0..4].try_into().unwrap()) as usize;
    &buf[4..4 + claimed_len]   // BAD: panics on claimed_len > buf.len() - 4 (DoS)
}
```

## Good (silent) shapes for the fixture

```rust
fn read_safe(buf: &[u8], i: usize) -> Option<u8> {
    buf.get(i).copied()
}

fn parse_chunk_safe(buf: &[u8]) -> Option<&[u8]> {
    if buf.len() < 4 { return None; }
    let claimed_len = u32::from_le_bytes(buf[0..4].try_into().ok()?) as usize;
    let end = 4_usize.checked_add(claimed_len)?;
    if end > buf.len() { return None; }
    Some(&buf[4..end])
}
```

## Composition note

Clippy's `clippy::indexing_slicing` covers `[i]` panics but NOT `unsafe` access, NOT `from_raw_parts`. This rule extends to those.
