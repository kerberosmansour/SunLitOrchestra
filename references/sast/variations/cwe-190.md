# Variation template — CWE-190 Integer Overflow / Wraparound

## Why this class

Rust release-mode wraps integer arithmetic by default unless `overflow-checks = true` is set. Debug panics; release silently wraps. The Rust Book explicitly says: "relying on this wrapping behaviour is considered an error."

Sink shapes:
- Capacity calculations (`Vec::with_capacity(a + b)`) where wrap → small allocation → subsequent OOB write.
- Length math (`buf.len() - offset`) where underflow → huge `usize` → OOB read in subsequent slice.
- Duration arithmetic (timestamp + delta) where wrap → past timestamp → bypasses expiry check.
- Casts that truncate (`u32 as u8`) where the high bits carry meaningful state.

## Sink shapes (4 minimum)

### 1. `capacity_calc_with_plus_or_mul`

```rust
fn alloc(header: usize, body: usize) -> Vec<u8> {
    Vec::with_capacity(header + body)   // BAD: wraps on overflow → small allocation
}
```

Use `checked_add` / `saturating_add`.

### 2. `length_subtract_unchecked`

```rust
fn tail(buf: &[u8], n: usize) -> &[u8] {
    &buf[buf.len() - n..]   // BAD: underflow if n > buf.len()
}
```

### 3. `duration_add_unchecked`

```rust
fn renew(now: u64, ttl: u64) -> u64 {
    now + ttl   // BAD: wrap → past timestamp → bypasses expiry check downstream
}
```

### 4. `cast_truncates_silently`

```rust
fn user_id_low_byte(id: u32) -> u8 {
    id as u8   // BAD: silently drops 24 bits; can collide when id > 255
}
```

## Good (silent) shapes for the fixture

```rust
fn alloc_safe(header: usize, body: usize) -> Option<Vec<u8>> {
    let cap = header.checked_add(body)?;
    Some(Vec::with_capacity(cap))
}

fn renew_safe(now: u64, ttl: u64) -> u64 {
    now.saturating_add(ttl)
}

fn user_id_low_byte_safe(id: u32) -> Result<u8, std::num::TryFromIntError> {
    u8::try_from(id)
}
```

## Composition note

Clippy's `clippy::arithmetic_side_effects` covers `+`, `-`, `*`, `/`, `%`, `<<` overflow + divide-by-zero panics broadly. **Allow by default** (restriction-group). The Semgrep rule narrows fire to security-sensitive contexts (capacity calc, length math, duration / timestamp arithmetic, casts) where the bug is most exploitable.
