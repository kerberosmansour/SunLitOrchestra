# Variation template — CWE-697 Incorrect Comparison

## Why this class

Comparing secrets / signatures / URLs / domains using the wrong predicate. Safe Rust does not protect: `==` on strings is byte-by-byte, fast-fail, and timing-side-channelable. URL / IDN / Punycode comparisons surface confusable inputs that compare-equal under one scheme and not another.

## Sink shapes (3 minimum)

### 1. `non_constant_time_secret_compare`

```rust
fn check_token(received: &str, expected: &str) -> bool {
    received == expected   // BAD: timing side-channel
}
```

Should use `subtle::ConstantTimeEq` or a pinned constant-time helper.

### 2. `punycode_or_idn_homograph_path`

```rust
fn allow_domain(host: &str) -> bool {
    host == "example.com"   // BAD: "ex‎ample.com" with a hidden Unicode char compares not-equal but renders identically
}
```

Should normalise via the `idna` crate before comparison.

### 3. `length_only_signature_check`

```rust
fn verify_sig(sig: &[u8], expected: &[u8]) -> bool {
    if sig.len() != expected.len() { return false; }
    sig.iter().zip(expected.iter()).all(|(a, b)| a == b)   // BAD: short-circuiting `all` is timing-side-channel
}
```

The `&&` short-circuit / `all` short-circuit equally betray secret length AND a leading-byte-match prefix.

## Good (silent) shapes for the fixture

```rust
use subtle::ConstantTimeEq;

fn check_token_safe(received: &[u8], expected: &[u8]) -> bool {
    received.ct_eq(expected).into()
}

fn allow_domain_safe(host: &str) -> bool {
    let ascii = idna::domain_to_ascii(host).unwrap_or_default();
    ascii == "example.com"
}
```

## Composition note

Clippy has no built-in constant-time-comparison lint. `subtle` is the canonical crate; `ring::constant_time::verify_slices_are_equal` is an alternative. The rule should NOT fire on these calls — only on the bare `==` / `iter().all`.
