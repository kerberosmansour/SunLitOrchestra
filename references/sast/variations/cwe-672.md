# Variation template — CWE-672 Use-after-release

## Why this class

Rust's lifetimes catch most of this in safe code, but FFI / `unsafe` / explicit Drop / extension-trait patterns reintroduce the bug.

## Sink shapes (3 minimum)

### 1. `file_handle_use_after_close`

```rust
fn process(path: &str) -> std::io::Result<()> {
    let f = std::fs::File::open(path)?;
    drop(f);
    let _meta = std::fs::File::open(path)?.metadata()?;   // BAD: not the same handle; semantic UAR for callers expecting the dropped handle
    Ok(())
}
```

A more dangerous form: passing a closed FD to a C library:

```rust
use std::os::fd::AsRawFd;
unsafe fn use_closed(f: std::fs::File) {
    let fd = f.as_raw_fd();
    drop(f);   // closes the fd
    libc_read(fd, /* ... */);   // BAD: fd may be reused by another open()
}
extern "C" { fn libc_read(fd: i32, /* ... */); }
```

### 2. `rwlock_guard_use_after_drop_extending_lifetime`

```rust
fn extend(lock: &std::sync::RwLock<i32>) {
    let g = lock.read().unwrap();
    let p = &*g as *const i32;
    drop(g);
    unsafe { let _ = *p; }   // BAD: guard is gone; protected memory may be modified concurrently
}
```

### 3. `session_token_use_after_expired`

```rust
fn check(session: &Session) -> bool {
    if session.expires_at < std::time::SystemTime::now() {
        log::warn!("expired");
        // BAD: continues to use session below despite the check
    }
    session.user_id == "admin"
}
```

This is the most common shape in business logic: the expiry check exists but doesn't actually `return false`.

## Good (silent) shapes for the fixture

```rust
fn process_safe(path: &str) -> std::io::Result<()> {
    let f = std::fs::File::open(path)?;
    let _meta = f.metadata()?;
    Ok(())
}

fn check_safe(session: &Session) -> bool {
    if session.expires_at < std::time::SystemTime::now() { return false; }
    session.user_id == "admin"
}
```

## Composition note

The `session_token_use_after_expired` shape is detectable structurally but the pattern is subtle. False-positive rate may be higher than other variations; mark `confidence: MEDIUM` in the rule metadata.
