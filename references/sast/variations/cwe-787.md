# Variation template — CWE-787 Out-of-bounds Write

## Why this class

The write-side of CWE-125. Same `unsafe` / FFI surface; same attacker-controlled length input. More dangerous than OOB read because the write can corrupt arbitrary memory.

## Sink shapes (3 minimum)

### 1. `vec_set_len_past_capacity`

```rust
fn build(buf: Vec<u8>, claimed: usize) -> Vec<u8> {
    let mut v = buf;
    unsafe { v.set_len(claimed); }   // BAD if claimed > v.capacity()
    v
}
```

### 2. `ptr_copy_nonoverlapping_attacker_len`

```rust
unsafe fn copy_in(dst: *mut u8, src: *const u8, n: usize) {
    std::ptr::copy_nonoverlapping(src, dst, n)   // BAD if n untrusted
}
```

### 3. `get_unchecked_mut_unchecked_index`

```rust
fn write(buf: &mut [u8], i: usize, b: u8) {
    unsafe { *buf.get_unchecked_mut(i) = b; }   // BAD: no bounds check
}
```

## Good (silent) shapes for the fixture

```rust
fn write_safe(buf: &mut [u8], i: usize, b: u8) -> Option<()> {
    *buf.get_mut(i)? = b;
    Some(())
}

fn build_safe(buf: Vec<u8>, claimed: usize) -> Option<Vec<u8>> {
    let mut v = buf;
    if claimed > v.capacity() { return None; }
    unsafe { v.set_len(claimed); }
    Some(v)
}
```

## Composition note

Same as CWE-125: Clippy's indexing-slicing covers safe path; this rule extends to `unsafe` and `ptr::copy*`. `miri` catches at runtime if a triggering test exists.
