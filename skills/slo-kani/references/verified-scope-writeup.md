# Verified-Scope Report — `docs/slo/verify/<slug>-kani.md`

The scope report is the durable, auditable record of what was proved. A green
Kani run is reported ONLY with this block. No whole-system claims. No
concurrency claims.

## Template

```markdown
# Kani Verified Scope — <slug>

> Generated by /slo-kani on <date>. Kani version: <pinned, from tools.toml>.
> A green run means "proved within the stated harness, assumptions, and bounds"
> — never "whole system proved."

## Properties proved
| Harness | Target fn | Property | Verdict |
|---|---|---|---|
| `check_foo` | `foo()` | no index-out-of-bounds | SUCCESSFUL |

## Proof scope (per harness)
| Harness | Bound (unwind / sizes) | Assumptions (`kani::assume`) | Stubs | Contracts | Solver |
|---|---|---|---|---|---|
| `check_foo` | `unwind(9)`, array len 8 | `len <= 8` | none | none | default |

## Catch -> remediate -> green (for fix loops)
| Harness | Pre-fix verdict (red) | Counterexample | Fix | Post-fix verdict (green) |
|---|---|---|---|---|

## What remains unproved
- <e.g. bounded at N=8 — says nothing about N>8 (disclosed, not eliminated)>
- <out-of-scope items: concurrency / interleavings -> /slo-tla>
```

## Rules

- Every green carries its bound, assumptions, stubs, and contracts. A green with
  no scope block is rejected.
- Each fixed bug shows its red (pre-fix) verdict before its green — the naive
  variant failed first.
- Out-of-scope features (concurrency, async, unbounded inputs) are listed
  explicitly under "What remains unproved", routed to `/slo-tla` where relevant.
- The Kani version is recorded so the result is reproducible against a pinned
  toolchain.
